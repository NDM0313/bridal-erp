### P0 - Critical (Fix Immediately)

**Criteria**: Blocks public launch, affects >50% users, data/security risk

**Examples**:
- [ ] Data loss bug
- [ ] Security vulnerability
- [ ] Payment processing broken
- [ ] Critical calculation error

**Process**:
1. Identify issue
2. Fix immediately
3. Test thoroughly
4. Deploy fix
5. Verify resolution
6. Restart 7-day clock if needed

**Timeline**: Fix within 24-48 hours

---

### P1 - High Priority (Fix Before Public Launch)

**Criteria**: Affects >25% users, significant UX issue, missing core feature

**Examples**:
- [ ] Confusing navigation flow
- [ ] Slow critical operation
- [ ] Missing essential feature
- [ ] Business logic issue

**Process**:
1. Prioritize fixes
2. Fix in order of impact
3. Test each fix
4. Deploy fixes
5. Re-test after fixes

**Timeline**: Fix within 1 week

---

### P2 - Medium Priority (Fix in Next Release)

**Criteria**: Affects <25% users, nice-to-have, performance optimization

**Examples**:
- [ ] Minor UX improvements
- [ ] Performance optimizations
- [ ] Nice-to-have features
- [ ] Enhancement requests

**Process**:
1. Add to backlog
2. Plan for next release
3. Implement when time permits
4. Test and deploy

**Timeline**: Fix in next release (2-4 weeks)

---

### P3 - Low Priority (Future Consideration)

**Criteria**: Nice-to-have, enhancement, future feature

**Examples**:
- [ ] Advanced features
- [ ] Integrations
- [ ] Customizations
- [ ] Future enhancements

**Process**:
1. Add to roadmap
2. Evaluate business value
3. Plan for future releases
4. Implement when prioritized

**Timeline**: Future releases
