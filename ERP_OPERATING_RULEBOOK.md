## Owner Daily Dashboard Guide

### Morning Routine (10-15 minutes)
1. **Open Dashboard** - Login, check today's date
2. **Check Sales Numbers** - Count, amount, compare with yesterday
3. **Check Production Status** - Dyeing, Handwork, Stitching counts
4. **Check Cash/Payments** - Paid, due, total due amount
5. **Check Alerts/Warnings** - Errors, warnings, notifications

### Mid-Day Quick Check (5 minutes)
1. **Sales are being created** - New sales since morning
2. **Production steps are being updated** - Step counts changed
3. **No errors reported** - Ask team if anyone faced problems

### End of Day Routine (10 minutes)
1. **Sales Check** - Review today's sales, verify finalized
2. **Production Check** - Review steps updated, costs entered
3. **Cash/Payment Check** - Review due amounts, plan collection
4. **Final Review** - Quick checklist, note issues for tomorrow

## Role-Wise Daily Checklists

### Production Worker
**Morning**: Login, check assigned steps  
**During Work**: Update quantity regularly, mark as in_progress when starting  
**End of Work**: Update final quantity, complete step if 100% done, logout

**Must Never Skip**: Login, quantity update, marking as in_progress, completing step

### Sales User
**Morning**: Login, check branch selection  
**During Work**: Select correct branch, add products correctly, create as draft first  
**Finalizing**: Review draft sale, finalize sale  
**End of Day**: Finalize all drafts, review today's sales, logout

**Must Never Skip**: Branch selection, finalizing sales, reviewing before finalizing

### Admin/Manager
**Morning**: Login, check dashboard numbers  
**During Day**: Monitor sales/production, enter costs promptly  
**Handling Exceptions**: Fix issues quickly, don't ignore problems  
**End of Day**: Review operations, verify data accuracy, check errors, logout

**Must Never Skip**: Dashboard check, cost entry, exception handling, data verification

**Full Guide**: See `ERP_ROLE_WISE_DAILY_CHECKLIST.md`

### Warning Sign #1: Users Not Logging In
**Action**: Identify problem → Address root cause → Enforce usage → Monitor

### Warning Sign #2: Users Reverting to Old System
**Action**: Stop old system → Address concerns → Enforce ERP-only → Verify compliance

### Warning Sign #3: Data Not Being Entered
**Action**: Identify missing data → Fix technical issues → Enforce data entry → Verify

### Warning Sign #4: Too Many Errors or Complaints
**Action**: Address critical errors → Categorize errors → Fix root causes → Verify stability

## Quick Reference

### Daily Numbers That Matter
- **Sales Count**: Should be normal (not zero)
- **Sales Amount**: Should match expectations
- **Production Steps Updated**: Should see progress
- **Payments Received**: Should match sales

### Quick Action Guide
- **If Sales Are Zero**: Check if sales staff is working, can login, know how to create sales
- **If Production Not Updating**: Check if workers are using mobile app, can login, steps assigned
- **If Errors Appear**: Read error message, note what happened, contact support if needed
- **If System is Slow**: Check internet, refresh, contact support if still slow

### Success Indicators
- **Daily**: Sales created, production progressing, payments recorded, no errors, team comfortable
- **Weekly**: Sales consistent, production on track, costs tracked, system stable, team confident
- **Monthly**: Business running smoothly, ERP is single source of truth, old system not needed, team fully adopted

## Emergency Procedures

### If System is Down
1. Stop creating new data (if possible)
2. Document what you were doing
3. Contact admin immediately
4. Wait for system recovery
5. Resume work after system is back

### If Data is Corrupted
1. Stop all operations
2. Contact admin immediately
3. Do NOT try to fix yourself
4. Wait for admin to restore from backup

### If Unauthorized Access Suspected
1. Change password immediately
2. Contact admin
3. Review recent activity
4. Report incident
