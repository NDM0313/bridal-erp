# Role-Wise Daily Checklist

**Simple Daily Tasks for Each Role**  
**Purpose**: Ensure ERP is used correctly every day

---

## A. Production Worker Daily Checklist

### Morning (Start of Work)
- [ ] **Login to mobile app**
  - Enter email and password
  - Verify you see "My Assigned Steps"
  - If you don't see steps, contact admin

- [ ] **Check assigned steps**
  - See how many steps you have
  - See which steps are pending
  - Plan your work for the day

**✅ Must Do**: Login and check steps every morning

---

### During Work
- [ ] **Update quantity as you work**
  - Don't wait until end of day
  - Update every 2-3 hours
  - Example: If you complete 20 units, update to 20

- [ ] **Mark step as "in_progress" when you start**
  - Don't leave steps in "pending" if you're working
  - Mark as "in_progress" when you begin

**✅ Must Do**: Update quantity regularly, mark as in_progress when starting

---

### End of Work
- [ ] **Update final quantity**
  - Enter exact quantity you completed today
  - Don't guess, count accurately

- [ ] **Complete step if 100% done**
  - Only mark as "completed" if work is fully done
  - Verify completed quantity = step quantity

- [ ] **Logout from app**
  - Prevents unauthorized access
  - Keeps your account secure

**✅ Must Do**: Update final quantity, complete if done, logout

---

### What Must Never Be Skipped
1. ❌ **Never skip login** - You must login to see your work
2. ❌ **Never skip quantity update** - System needs accurate progress
3. ❌ **Never skip marking as in_progress** - Others need to know you're working
4. ❌ **Never skip completing step** - Production order won't complete without it

---

## B. Sales User Daily Checklist

### Morning (Start of Work)
- [ ] **Login to system**
  - Enter email and password
  - Verify you see "Create Sale" option
  - If you don't see it, contact admin

- [ ] **Check branch selection**
  - Verify correct branch is selected in header
  - Don't create sale in wrong branch
  - If wrong branch, switch before creating sale

**✅ Must Do**: Login and verify branch every morning

---

### During Work (Creating Sales)
- [ ] **Select correct branch**
  - Check branch name in header
  - Don't create sale in wrong branch

- [ ] **Add products correctly**
  - Select correct product
  - Enter correct quantity
  - Verify stock is available

- [ ] **Select customer (if applicable)**
  - Link sale to customer for tracking
  - Use "Walk-in Customer" if no customer

- [ ] **Create sale as draft first**
  - Review before finalizing
  - Make sure everything is correct

**✅ Must Do**: Select correct branch, add products correctly, create as draft first

---

### Finalizing Sales
- [ ] **Review draft sale**
  - Check products are correct
  - Check quantities are correct
  - Check customer is correct

- [ ] **Finalize sale**
  - Click "Finalize Sale" button
  - Wait for success message
  - Verify sale appears in sales list

**✅ Must Do**: Review before finalizing, finalize immediately after review

---

### End of Day
- [ ] **Finalize all draft sales**
  - Don't leave drafts overnight
  - Finalize all sales before leaving

- [ ] **Review today's sales**
  - Check sales list
  - Verify all sales are finalized
  - Check for any errors

- [ ] **Logout from system**
  - Prevents unauthorized access
  - Keeps your account secure

**✅ Must Do**: Finalize all drafts, review sales, logout

---

### Handling WhatsApp Orders
- [ ] **When customer sends WhatsApp message**
  - Admin will create lead from message
  - You will see lead in system (if configured)

- [ ] **Create sale from lead**
  - Select customer (from WhatsApp number)
  - Add products customer wants
  - Create sale as normal

- [ ] **Finalize sale**
  - Review and finalize
  - Customer will receive confirmation (if configured)

**✅ Must Do**: Handle WhatsApp orders same as regular orders

---

### What Must Never Be Skipped
1. ❌ **Never skip branch selection** - Wrong branch = wrong inventory
2. ❌ **Never skip finalizing sales** - Drafts don't deduct stock
3. ❌ **Never skip reviewing before finalizing** - Mistakes are hard to fix
4. ❌ **Never skip customer selection** - Needed for proper tracking

---

## C. Admin/Manager Daily Checklist

### Morning (Start of Work)
- [ ] **Login to system**
  - Enter email and password
  - Go to Dashboard

- [ ] **Check dashboard numbers**
  - Sales count and amount
  - Production step counts
  - Payment status
  - Any errors or warnings

**✅ Must Do**: Login and check dashboard every morning

---

### During Day (Monitoring)
- [ ] **Monitor sales activity**
  - Check if sales are being created
  - Check if sales are being finalized
  - Verify sales look correct

- [ ] **Monitor production progress**
  - Check if steps are being updated
  - Check if steps are being completed
  - Verify production is on track

- [ ] **Enter production costs**
  - When step is completed, enter cost
  - Don't delay cost entry
  - Verify cost is correct

**✅ Must Do**: Monitor activity, enter costs promptly

---

### Handling Exceptions
- [ ] **If worker cannot update step**
  - Check if step is assigned to correct worker
  - Check if worker has correct role
  - Verify step status

- [ ] **If sale not creating production order**
  - Check if product has "requires_production" flag
  - Check if sale status is "final"
  - Verify production order was created

- [ ] **If cost not creating expense**
  - Check if step cost > 0
  - Check if step status is "completed"
  - Verify expense was created

**✅ Must Do**: Handle exceptions quickly, don't ignore problems

---

### End of Day
- [ ] **Review daily operations**
  - Check sales summary
  - Check production summary
  - Check payment summary

- [ ] **Verify data accuracy**
  - Compare with expectations
  - Check for discrepancies
  - Verify everything looks correct

- [ ] **Check for errors**
  - Review error logs (if accessible)
  - Check for user-reported issues
  - Plan fixes for tomorrow

- [ ] **Logout from system**
  - Prevents unauthorized access
  - Keeps account secure

**✅ Must Do**: Review operations, verify data, check errors, logout

---

### Overrides (When Allowed)
- [ ] **Production step assignment**
  - Can assign step to different worker if needed
  - Document reason for change

- [ ] **Production step cost**
  - Can update cost if entered incorrectly
  - Verify cost is correct

- [ ] **User roles**
  - Can change user roles if needed
  - Document reason for change

**✅ Must Do**: Only override when necessary, document reason

---

### What Must Never Be Skipped
1. ❌ **Never skip dashboard check** - Need to monitor business daily
2. ❌ **Never skip cost entry** - Costs must be tracked
3. ❌ **Never skip exception handling** - Problems get worse if ignored
4. ❌ **Never skip data verification** - Accuracy is critical

---

## General Rules for All Roles

### Must Do Every Day
1. ✅ **Login** - Start every day with login
2. ✅ **Complete your tasks** - Don't leave work incomplete
3. ✅ **Logout** - End every day with logout
4. ✅ **Report issues** - Don't suffer in silence

### Must Never Do
1. ❌ **Don't share passwords** - Each user must have own account
2. ❌ **Don't skip steps** - Follow checklist every day
3. ❌ **Don't ignore errors** - Report problems immediately
4. ❌ **Don't use old system** - ERP is single source of truth
